Soular Souls X-VP SDK v.01 Release Notes

The Soular Souls X-VP SDK provides developers with documentation, header files, and sample code necessary to write software using X-VP. The X-VP SDK Installer simplifies installation by installing the entire X-VP source tree (depending on license type) and all sample code, tutorials and other documentation to your hard disk.  It creates several convenient shortcuts on your Windows Start Menu to enable quick access to key components of the installation.


What's New in v.01?

This is the first public beta of the X-VP SDK.


File System Layout

By default, the X-VP SDK is installed to your hard disk in the locations described in the following table. Keep in mind, you may or may not have all the directories listed in the table below depending on the license type you've been granted.

Directory Contents
\bin Executable utilities for the X-VP SDK
\dll-debug Debug X-VP DLL build directory
\dll-release Release X-VP DLL build directory
\exe-debug Debug X-VP .exe build directory. Used for temporary development
\exe-release Release X-VP .exe build directory. Used for temporary development
\lib-debug Debug X-VP static lib build directory
\lib-release Release X-VP static lib build directory
\inc Various header files needed to build X-VP applications
\src Source files for X-VP
\lib Library files needed to link X-VP applications
\tutorials The X-VP tutorial set including code samples
\tools Tools associated with X-VP
\exporters Various content exporters for X-VP
\docs Contains all X-VP documentation
\scrapbook Old items that are not compiled or used, but may be useful in the future
\unsupported Unsupported code and miscellaneous items


Prerelease Content

This version of the X-VP SDK includes prerelease content, enabling you to start developing your application before the technology is released. However, technologies can change prior to their final release. Therefore, you may want to wait until the SDK reaches version 1.0, which will be considered the first stable and fully supported commercial release.  If you decide to release your commercial application prior to version 1.0, discounted licenses may be obtained, please see http://www.soularsouls.com for the current offerings. 

For more information, see the license included with this SDK distribution. Additionally, some material is a preview of new or planned functionality not yet available in the SDK.


Building Applications

The details of configuring compiler and linker options are described in the Getting Started Guide.  The X-VP SDK may require other 3rd party SDKs to be installed in order to function properly such as Microsoft's DirectX or OpenAL.  Please see the Getting Started Guide for more information on the setup required for your particular environment. 
 

Tested Compilers/Build Environments

The X-VP SDK has been tested with Visual Studio .NET.  Over time, we will extend support for other compilers as users will inevitably try others during development.  If you are using another system successfully, please let us know so we can add it to the list of supported compilers.  Send email to support@soularsouls.com.  For more information about Visual Studio, see the Microsoft Visual Studio home page.


Development Notes

  • Thus far we've only tested 32-bit Windows Systems, but we don't suspect any 64-bit issues.  Until such time that X-VP is targeted towards 64-bit operating systems, we cannot officially support X-VP development on 64-bit systems. 
  • At this time, only the Windows XSP is available.


Tutorials and Sample Code

The X-VP SDK includes several code examples that demonstrate how to use X-VP.  A quick way to access these tutorials is via the Start Menu after X-VP installation.  The tutorials start with simple concepts and progress towards more advanced practices.  We recommend you start out at the beginning of the tutorials and work your way through them one by one making sure you understand the concepts presented in each one as later tutorials may be combinations of ideas presented in earlier tutorials.

Keep in mind as you work through the sample code that clarity and ease of explanation were always chosen over optimization and algorithm choice.  That means that a tutorial's main goal is to explain the specific topic at hand without clouding the code with excessive error handling or optimizations that might make the code less straight forward.
